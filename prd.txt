  코드 설명 방식 (Claude에게)
----------------------------------------------------------------

  "prd에 기록된 설명방식 인지하고 그렇게 설명해주세요" 라고 하면
  아래 방식으로 설명한다.

  ✅ 설명 규칙
      - 흐름 순서대로 한 단계씩, 사용자가 "다음"이라고 하면 진행
      - 파일명은 클릭 가능한 마크다운 링크로 표시
        예) [application.properties](back-end/src/main/resources/application.properties)
      - 코드 블록으로 실제 코드를 직접 보여주면서 설명
      - 각 설정/코드 줄이 왜 있는지 한 줄씩 한글로 설명
      - 전문 용어는 쉬운 말로 풀어서 설명
        예) "classpath = src/main/resources 폴더를 가리킴"
      - 단계 마지막에는 항상 "이해되셨으면 다음 이라고 해주세요" 안내

----------------------------------------------------------------
  코드 작성 원칙 (반드시 준수)
----------------------------------------------------------------

  이 프로젝트의 코드는 개발을 처음 해보는 초보 대학생이 직접
  짠 것처럼 작성되어야 합니다.

  ✅ 해야 할 것
      - 변수명, 함수명은 누가 봐도 바로 이해되는 직관적인 이름
      - 복잡한 패턴보다 단순하고 직선적인 코드
      - 한글 주석으로 각 코드가 무슨 역할인지 설명
      - 기능별로 함수를 나누되 과도하게 쪼개지 않음
      - 초보자가 실제로 짤 법한 구조와 흐름 유지

  ❌ 하지 말 것
      - useCallback, useMemo 같은 고급 최적화 패턴
      - 과도한 추상화, 불필요한 컴포넌트 분리
      - 전문 개발자 스타일의 복잡한 설계
      - 설명 없이 이해하기 어려운 단축 문법
      - @RequiredArgsConstructor (Lombok 생성자 주입) 사용 금지
        → 생성자를 직접 작성할 것 (IDE가 Java 25로 인식해 Lombok 충돌)
      - @Slf4j 사용 금지
        → private static final Logger log = LoggerFactory.getLogger(XXX.class); 직접 선언

----------------------------------------------------------------
  기술 스택
----------------------------------------------------------------
  백엔드 : Java 22 / Spring Boot 3.3.6 / Spring Security + 세션
          MariaDB 12.2 / Spring Data JPA (ddl-auto=none)
          Quartz Scheduler / Maven
  프론트 : 바닐라 JS + Bootstrap 5 CDN + FullCalendar 6 CDN
          (Spring Boot static 폴더에서 서빙 — 서버 하나)
  외부 API: 한국관광공사 KorService2 (공공데이터포털)
           한국관광공사 EngService2 (영문 — API 키 승인 대기 중)
  DB 테이블: users, event, user_schedule, event_translation

----------------------------------------------------------------
  실행 방법
----------------------------------------------------------------
  cd back-end
  JAVA_HOME="C:/Program Files/Java/jdk-22" ./mvnw spring-boot:run

  ※ clean이 필요할 때 (새 파일 추가 후 첫 실행):
  JAVA_HOME="C:/Program Files/Java/jdk-22" ./mvnw clean spring-boot:run

  ※ 한글 로그 깨질 때:
  JAVA_HOME="C:/Program Files/Java/jdk-22" ./mvnw spring-boot:run -Dspring-boot.run.jvmArguments="-Dfile.encoding=UTF-8"

  → http://localhost:8080 접속

----------------------------------------------------------------
  주요 결정 사항
----------------------------------------------------------------
  ✅ 프론트엔드: 바닐라 JS (React 제거), static 폴더 서빙
  ✅ 인증: JWT 제거 → Spring Session (JSESSIONID 쿠키 자동 관리)
  ✅ DB 테이블 생성: data.sql (CREATE TABLE IF NOT EXISTS)
     spring.sql.init.mode=always 로 서버 시작 시 자동 실행
  ✅ API 키: .env 파일에 TOUR_API_KEY=실제키 저장 (gitignore 등록)
     application.properties → api.tour.key=${TOUR_API_KEY:test_api_key}

----------------------------------------------------------------
  백엔드
----------------------------------------------------------------

  ✅ Spring Boot 3.3.6 프로젝트 세팅
  ✅ MariaDB 연결 (local-connect 데이터베이스, root/1234)

  ── 인증 (auth 패키지) ──────────────────────────────────────
  ✅ User 엔티티 (users 테이블)
     - id, email(unique), password(bcrypt), nickname(unique), created_at
  ✅ POST /api/auth/signup   → 회원가입 (email, password, nickname)
  ✅ POST /api/auth/login    → 로그인 (세션 생성)
  ✅ POST /api/auth/logout   → 로그아웃 (세션 삭제)
  ✅ GET  /api/auth/me       → 로그인 유저 정보 반환 { email, nickname }
  ✅ Spring Security 세션 인증
     - CSRF 비활성화 (REST API 방식)
     - /api/auth/**, /api/events/** → permitAll
     - /api/schedules/** → 로그인 필요

  ── 축제 (event 패키지) ─────────────────────────────────────
  ✅ Event 엔티티 (event 테이블)
     - id, contentId(unique), title, startDate, endDate
       address, region, imageUrl, tel, latitude, longitude, updatedAt
  ✅ GET  /api/events                → 목록 (페이징, 지역 필터, 키워드 검색)
     GET  /api/events?region=서울    → 지역 필터
     GET  /api/events?keyword=벚꽃  → 키워드 검색
     GET  /api/events?lang=en       → 영문 제목으로 반환 (번역 없으면 한국어 폴백)
     GET  /api/events?page=0&size=12&sort=startDate,asc → 페이지네이션
  ✅ GET  /api/events/{id}           → 상세 조회 (overview, homepage 포함)
     GET  /api/events/{id}?lang=en  → 영문 상세 조회
  ✅ POST /api/events/sync           → 한국어 데이터 수동 동기화
  ✅ POST /api/events/sync/en        → 영문 데이터 수동 동기화
     ※ EngService2는 현재 403 (API 키 미승인) — 팀원이 신청 중
  ✅ TourApiClient: KorService2 + EngService2 호출
     - fetchFestivals(areaCode, eventStartDate) → 한국어 목록
     - fetchFestivalsEng(areaCode, eventStartDate) → 영문 목록
     - fetchDetail(contentId) → 한국어 상세 (overview, homepage)
     - fetchDetailEng(contentId) → 영문 상세
  ✅ Quartz 스케줄러 (매일 새벽 2시)
     → syncAll() (한국어) 후 syncEnglish() (영문) 순서로 실행
  ✅ 축제 데이터 수량 문제 수정
     - eventStartDate를 "오늘"→"올해 1월 1일"로 변경 (진행 중 축제 포함)
     - numOfRows 100→500으로 변경 (지역별 축제 전부 가져오기)
  ✅ EventTranslation 엔티티 (event_translation 테이블)
     - content_id, language('en'), title
     - EventTranslationRepository: findByContentIdAndLanguage()

  ── 일정 (schedule 패키지) ──────────────────────────────────
  ✅ UserSchedule 엔티티 (user_schedule 테이블)
     - id, userId, eventId, eventTitle, startDate, endDate
       region, imageUrl, savedAt
  ✅ POST   /api/schedules      → 일정 저장
     - body: { eventId, startDate, endDate }
     - startDate/endDate: 사용자가 달력에서 드래그 선택한 날짜
     - 없으면 축제 전체 기간 그대로 저장
     - 같은 축제 중복 저장 불가
  ✅ DELETE /api/schedules/{id} → 일정 삭제 (본인 것만)
  ✅ GET    /api/schedules      → 내 일정 목록

----------------------------------------------------------------
  프론트엔드 (static 폴더)
----------------------------------------------------------------

  ✅ index.html — 메인 페이지
     - Bootstrap 5 CDN + FullCalendar 6 CDN
     - 상단 navbar: 로고, 내 일정 링크, 언어 전환 버튼(English/한국어), 로그인 상태
     - 검색창, 지역 필터 버튼(전체 + 17개 지역)
     - 축제 카드 그리드 (12개씩)
     - 페이지네이션
     - 축제 상세 모달 (#festivalModal)
     - 날짜 선택 모달 (#schedulePickerModal) — FullCalendar
     - 로그인/회원가입 모달 (#authModal)

  ✅ js/main.js — 메인 페이지 로직
     주요 전역 변수:
       currentPage, currentRegion, currentKeyword, totalPages
       currentTab ('login'|'signup')
       currentUser (로그인 유저 정보 또는 null)
       pickerCalendar (FullCalendar 인스턴스)
       currentFestivalData (현재 열린 축제 데이터)
       selectedScheduleStart, selectedScheduleEnd (드래그 선택 날짜)
       currentLang ('ko'|'en')

     주요 함수:
       checkLoginStatus()      → GET /api/auth/me → currentUser 세팅
       updateAuthUI()          → 헤더 로그인/로그아웃 버튼 업데이트
       renderRegionButtons()   → 지역 필터 버튼 생성
       selectRegion(region)    → 지역 선택
       handleSearch(e)         → 검색
       loadFestivals()         → GET /api/events?lang=${currentLang}&...
       renderFestivals(list)   → 카드 그리기
       renderPagination()      → 페이지 버튼
       openFestivalModal(id)   → GET /api/events/{id}?lang=${currentLang} → 모달
       openSchedulePicker(f)   → 날짜 선택 모달 열기
       initPickerCalendar(f)   → FullCalendar 초기화
         - selectable: true
         - selectAllow: 축제 기간 내만 선택 허용
         - dayCellClassNames: 기간 외 날짜 .day-unavailable (회색)
         - select: 드래그 선택 → selectedScheduleStart/End 업데이트
       confirmSaveSchedule()   → POST /api/schedules
       toggleLanguage()        → currentLang 전환 후 loadFestivals()
       handleAuth(e)           → 로그인/회원가입 폼 제출
       handleLogout()          → POST /api/auth/logout
       formatDate(str)         → "20250501" → "2025년 5월 1일"
       toCalendarDate(str)     → "20250501" → "2025-05-01"
       addOneDay(str)          → FullCalendar 종료일 +1일

  ✅ calendar.html — 내 일정 페이지
  ✅ js/calendar.js — 내 일정 로직
     - checkLoginStatus() → 로그인 안됐으면 안내 메시지
     - loadSchedules()    → GET /api/schedules
     - renderCalendar()   → FullCalendar (dayGridMonth, listMonth)
     - renderScheduleList() → 일정 카드 목록
     - deleteSchedule(id) → DELETE /api/schedules/{id}

  ✅ css/style.css
     - .festival-card hover 효과 (translateY, box-shadow)
     - .day-unavailable { background: #e9ecef } — 선택 불가 날짜 회색
     - .fc-highlight { background: rgba(144,238,144,0.5) } — 선택 날짜 녹색
