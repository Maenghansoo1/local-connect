================================================================

  이 프로젝트는 개발을 처음 해보는 초보 대학생이 직접 작성한
  코드입니다. 아래 원칙을 절대 어기지 마세요.

  1. 코드는 항상 초보 대학생 수준으로 작성한다.
     - 누가 봐도 바로 이해되는 변수명/함수명
     - 한글 주석으로 각 코드의 역할 설명
     - 단순하고 직선적인 구조 (복잡한 패턴, 과도한 추상화 금지)

  2. 중간에 작업이 중단되어도 학생이 직접 이어서 개발할 수 있도록
     고차원적인 구조로 코드를 작성하지 않는다.
     - 전문 개발자만 이해하는 설계 금지
     - AI가 멈춘 자리에서 학생이 직접 이어 쓸 수 있는 수준 유지

  3. 컴팩(context compaction) 후 새 대화가 시작되면:
     → prd.txt 전체를 먼저 읽는다
     → 관련 코드 파일을 읽는다
     → 그 다음 작업을 이어한다
     → 사용자에게 "이전 작업 내용을 파악했습니다" 라고 알린다

  4. 코드 작성 주의사항
     - 현재 프로젝트는 Lombok 사용 안 함 → getter/setter, 생성자 직접 작성
     - DB 접근은 MyBatis(@Mapper + @Select/@Insert 등 어노테이션 SQL) 사용
       (JPA 아님 — 팀원이 JPA에서 MyBatis로 전환함)
     - 테이블은 schema.sql 로 자동 생성됨 (JPA ddl-auto 아님)

  5. 힌트 참고:
     - mid-develop 브랜치에 이전 Bootstrap + 축제DB 버전 있음
       (구조가 완전히 달라서 직접 이식은 불가, UI/아이디어 참고용)
     - 영문 번역, 일정 저장, 세션 인증 등 참고 가능

----------------------------------------------------------------
  프로젝트 개요
----------------------------------------------------------------

  이름: Local Connect
  목적: 한국 관광지를 지역·카테고리별로 탐색하고, 즐겨찾기·리뷰·
        방문 기록을 관리할 수 있는 한/영 이중 언어 웹 서비스

----------------------------------------------------------------
  기술 스택
----------------------------------------------------------------

  [백엔드]
  - Java 21, Spring Boot 3.5.14
  - Spring Security (세션 기반 로그인)
  - MyBatis (@Mapper 인터페이스에 SQL 직접 작성, JPA 아님)
  - MariaDB (로컬 DB), 테이블은 schema.sql 로 자동 생성
  - spring-dotenv (.env 파일로 API 키·비밀번호 관리)
  - 한국관광공사 API (KorService2 한국어, EngService2 영어)

  [프론트엔드]
  - Vanilla JS (React 같은 프레임워크 없이 순수 JS)
  - Kakao Maps API (지도, 마커, 카카오 내비게이션)
  - FullCalendar v6 (여행 일정 달력)
  - 한/영 i18n 지원 (i18n.js)

  [기타]
  - WebSocket 의존성 추가됨 (현재 미사용, 향후 기능 예정)

----------------------------------------------------------------
  DB 테이블 구조
----------------------------------------------------------------

  [users] — 회원 정보
  id, username(unique), password(BCrypt 암호화), nickname,
  email(unique), provider, providerId, createdAt

  [reviews] — 관광지 리뷰
  id, user_id(FK→users), contentId, spotTitle,
  content(최대 1000자), rating(1~5), createdAt

  [favorites] — 즐겨찾기
  id, user_id(FK→users), contentId, title, addr,
  image, contentTypeId, mapx, mapy, createdAt

  [visit_history] — 방문 기록
  id, user_id(FK→users), contentId, title, addr, image, visitedAt

  [schedules] — 여행 일정 (달력)
  id, user_id(FK→users), title, travel_date(여행 날짜),
  memo, created_at

  [schedule_items] — 일정 안에 담긴 장소들
  id, schedule_id(FK→schedules), contentId, title, addr,
  visit_time, item_order(순서), memo

  ※ 테이블은 schema.sql 로 자동 생성 (CREATE TABLE IF NOT EXISTS)
  ※ DB 컬럼은 snake_case, 자바 필드는 camelCase
     (application.properties 의 map-underscore-to-camel-case=true 로 자동 매핑)

----------------------------------------------------------------
  API 엔드포인트 목록
----------------------------------------------------------------

  인증 (비로그인 접근 가능)
  POST   /api/auth/signup              — 회원가입
  POST   /api/auth/login               — 로그인 (username/password)
  POST   /api/auth/logout              — 로그아웃
  GET    /api/auth/me                  — 현재 로그인 유저 확인

  관광지 (비로그인 접근 가능)
  GET    /api/spots                    — 관광지 목록 (?areaCode=&contentTypeId=&lang=&pageNo=&numOfRows=)
  GET    /api/spots/common             — 관광지 소개 정보 (?contentId=&lang=)
                                          → detailCommon2 API (소개글·전화·홈페이지·주소)
  GET    /api/spots/detail             — 관광지 상세 (?contentId=&contentTypeId=&lang=)
                                          → detailIntro2 API (축제 기간/장소 등)

  리뷰
  POST   /api/reviews                  — 리뷰 등록 (로그인 필요)
  GET    /api/reviews/spot/{contentId} — 특정 관광지 리뷰 목록 (비로그인 가능)
  GET    /api/reviews/my               — 내 리뷰 목록 (로그인 필요)
  DELETE /api/reviews/{reviewId}       — 리뷰 삭제 (본인만 가능)

  즐겨찾기 (로그인 필요)
  POST   /api/favorites/toggle         — 즐겨찾기 추가/취소
  GET    /api/favorites                — 즐겨찾기 목록
  GET    /api/favorites/check          — 즐겨찾기 여부 확인 (?contentId=)

  방문 기록 (로그인 필요)
  POST   /api/visits                   — 방문 기록 저장 (카드 클릭 시 자동)
  GET    /api/visits                   — 방문 기록 목록

  여행 일정 / 달력 (로그인 필요)
  GET    /api/schedules                — 내 일정 전체 목록 (장소 포함)
  POST   /api/schedules                — 새 일정 생성 (title, travelDate, memo)
  GET    /api/schedules/{id}           — 일정 상세 (장소 포함)
  DELETE /api/schedules/{id}           — 일정 삭제 (장소도 함께 삭제)
  POST   /api/schedules/{id}/items     — 일정에 장소 추가
  DELETE /api/schedules/{id}/items/{itemId} — 일정에서 장소 삭제

----------------------------------------------------------------
  구현 완료된 기능
----------------------------------------------------------------

  ✅ 회원 관리
     - 회원가입 / 로그인 / 로그아웃
     - BCrypt 비밀번호 암호화
     - 세션 기반 인증 (Spring Security Form Login)

  ✅ 관광지 탐색
     - 지역 선택 (17개 시/도)
     - 카테고리 선택 (관광지/문화시설/축제/레포츠/숙박/쇼핑/음식점)
     - 한국관광공사 API로 목록·상세 조회 (한/영)
     - 목록 페이징 (한 페이지 6개, 페이지 버튼)
     - 축제(15) 선택 시 기간/장소/홈페이지 상세 표시
     - 카카오맵 마커 표시 + 인포윈도우 클릭
     - 카카오 내비게이션 연결 (새 탭)

  ✅ 관광지 상세정보 모달 (spot-info.js)
     - 카드의 사진/제목 클릭 시 상세정보 창 열림
     - detailCommon2 API로 주소·전화·홈페이지·소개글 표시
     - 리뷰 모달(💬 리뷰 보기)과는 별개의 창

  ✅ 여행 일정 / 달력 (calendar.js, schedule 패키지)
     - FullCalendar 달력에서 날짜 클릭 → 그 날 일정 표시
     - 일정 생성/삭제, 일정에 관광지 장소 추가/삭제
     - 관광지 상세에서 "일정에 추가" 버튼으로 장소 담기

  ✅ 즐겨찾기
     - 카드에서 하트 버튼으로 토글 (❤️/🤍)
     - 마이페이지에서 즐겨찾기 목록 확인

  ✅ 리뷰
     - 관광지 상세 모달에서 별점(1~5) + 내용 작성
     - 비로그인도 리뷰 목록 조회 가능
     - 마이페이지에서 내 리뷰 확인 및 삭제

  ✅ 방문 기록
     - 관광지 카드 클릭 시 자동으로 저장 (로그인 상태일 때만)
     - 마이페이지에서 방문 기록 날짜 순으로 확인

  ✅ 다국어 지원
     - 한국어 / 영어 전환 (i18n.js)
     - 영어 선택 시 한국관광공사 영문 API (EngService2) 호출

  ※ spring-dotenv 라이브러리가 자동으로 읽어서 application.properties에 주입

----------------------------------------------------------------
  실행 방법
----------------------------------------------------------------

  1. MariaDB 실행 후 데이터베이스 생성:
     CREATE DATABASE projectDB CHARACTER SET utf8mb4;
     (테이블은 서버 실행 시 schema.sql 이 자동 생성)

  2. 프로젝트 루트에 .env 파일 생성 후 API 키, DB 비밀번호 입력

  3. 터미널에서 실행:
     ./mvnw spring-boot:run      ← Git Bash 에서는 이 명령어
     mvnw.cmd spring-boot:run    ← cmd(명령 프롬프트) 에서는 이 명령어
     ※ Java 파일을 수정하면 서버를 껐다(Ctrl+C) 다시 켜야 반영됨
     ※ HTML/CSS/JS 만 고쳤으면 브라우저 강력 새로고침(Ctrl+F5)만 하면 됨

  4. 브라우저에서 접속:
     http://localhost:8080
