  이 프로젝트는 개발을 처음 해보는 초보 대학생이 직접 작성한
  코드입니다. 아래 원칙을 절대 어기지 마세요.

  1. 코드는 항상 초보 대학생 수준으로 작성한다.
     - 누가 봐도 바로 이해되는 변수명/함수명
     - 한글 주석으로 각 코드의 역할 설명
     - 단순하고 직선적인 구조 (복잡한 패턴, 과도한 추상화 금지)

  2. 중간에 작업이 중단되어도 학생이 직접 이어서 개발할 수 있도록
     고차원적인 구조로 코드를 작성하지 않는다.
     - 전문 개발자만 이해하는 설계 금지
     - AI가 멈춘 자리에서 학생이 직접 이어 쓸 수 있는 수준 유지

  3. 컴팩(context compaction) 후 새 대화가 시작되면:
     → prd.txt 전체를 먼저 읽는다
     → 관련 코드 파일을 읽는다
     → 그 다음 작업을 이어한다
     → 사용자에게 "이전 작업 내용을 파악했습니다" 라고 알린다

  4. 팀원 코드 주의사항 (backend/ 폴더)
     - @RequiredArgsConstructor, @Slf4j 등 Lombok 이미 사용 중 → 그대로 유지
     - 단, 새 파일 추가 시에는 직접 생성자/Logger 작성
     - H2 인메모리 DB 사용 (MariaDB 아님)

  5. 힌트 참고:
     - mid-develop 브랜치에 이전 바닐라JS + MariaDB 버전 있음
     - 영문 번역, 일정 저장, 세션 인증 등 참고 가능

  코드 설명 방식 (Claude에게)
----------------------------------------------------------------

  "prd에 기록된 설명방식 인지하고 그렇게 설명해주세요" 라고 하면
  아래 방식으로 설명한다.

  ✅ 설명 규칙
      - 흐름 순서대로 한 단계씩, 사용자가 "다음"이라고 하면 진행
      - 파일명은 클릭 가능한 마크다운 링크로 표시
        예) [application.yml](backend/src/main/resources/application.yml)
      - 코드 블록으로 실제 코드를 직접 보여주면서 설명
      - 각 설정/코드 줄이 왜 있는지 한 줄씩 한글로 설명
      - 전문 용어는 쉬운 말로 풀어서 설명
      - 단계 마지막에는 항상 "이해되셨으면 다음 이라고 해주세요" 안내

----------------------------------------------------------------
  프로젝트 개요
----------------------------------------------------------------

  이름: Local Connect
  목적: 한국 관광지를 지역·카테고리별로 탐색하고, 즐겨찾기·리뷰·
        방문 기록을 관리할 수 있는 한/영 이중 언어 웹 서비스

----------------------------------------------------------------
  기술 스택
----------------------------------------------------------------

  [백엔드]
  - Java 21, Spring Boot 3.5.14
  - Spring Security (세션 기반 로그인)
  - Spring Data JPA (ORM, 객체로 DB 조작)
  - MariaDB (로컬 DB)
  - spring-dotenv (.env 파일로 API 키·비밀번호 관리)
  - 한국관광공사 API (KorService2 한국어, EngService2 영어)

  [프론트엔드]
  - Vanilla JS (React 같은 프레임워크 없이 순수 JS)
  - Kakao Maps API (지도, 마커, 카카오 내비게이션)
  - 한/영 i18n 지원 (i18n.js)

  [기타]
  - WebSocket 의존성 추가됨 (현재 미사용, 향후 기능 예정)

----------------------------------------------------------------
  DB 테이블 구조
----------------------------------------------------------------

  [users] — 회원 정보
  id, username(unique), password(BCrypt 암호화), nickname,
  email(unique), provider, providerId, createdAt

  [reviews] — 관광지 리뷰
  id, user_id(FK→users), contentId, spotTitle,
  content(최대 1000자), rating(1~5), createdAt

  [favorites] — 즐겨찾기
  id, user_id(FK→users), contentId, title, addr,
  image, contentTypeId, mapx, mapy, createdAt

  [visit_history] — 방문 기록
  id, user_id(FK→users), contentId, title, addr, image, visitedAt

  ※ JPA가 ddl-auto=update 설정으로 테이블 자동 생성/수정

----------------------------------------------------------------
  API 엔드포인트 목록
----------------------------------------------------------------

  인증 (비로그인 접근 가능)
  POST   /api/auth/signup              — 회원가입
  POST   /api/auth/login               — 로그인 (username/password)
  POST   /api/auth/logout              — 로그아웃
  GET    /api/auth/me                  — 현재 로그인 유저 확인

  관광지 (비로그인 접근 가능)
  GET    /api/spots                    — 관광지 목록 (?areaCode=&contentTypeId=&lang=)
  GET    /api/spots/detail             — 관광지 상세 (?contentId=&contentTypeId=&lang=)

  리뷰
  POST   /api/reviews                  — 리뷰 등록 (로그인 필요)
  GET    /api/reviews/spot/{contentId} — 특정 관광지 리뷰 목록 (비로그인 가능)
  GET    /api/reviews/my               — 내 리뷰 목록 (로그인 필요)
  DELETE /api/reviews/{reviewId}       — 리뷰 삭제 (본인만 가능)

  즐겨찾기 (로그인 필요)
  POST   /api/favorites/toggle         — 즐겨찾기 추가/취소
  GET    /api/favorites                — 즐겨찾기 목록
  GET    /api/favorites/check          — 즐겨찾기 여부 확인 (?contentId=)

  방문 기록 (로그인 필요)
  POST   /api/visits                   — 방문 기록 저장 (카드 클릭 시 자동)
  GET    /api/visits                   — 방문 기록 목록

----------------------------------------------------------------
  구현 완료된 기능
----------------------------------------------------------------

  ✅ 회원 관리
     - 회원가입 / 로그인 / 로그아웃
     - BCrypt 비밀번호 암호화
     - 세션 기반 인증 (Spring Security Form Login)

  ✅ 관광지 탐색
     - 지역 선택 (17개 시/도)
     - 카테고리 선택 (관광지/문화시설/축제/레포츠/숙박/쇼핑/음식점)
     - 한국관광공사 API로 목록·상세 조회 (한/영)
     - 축제(15) 선택 시 기간/장소/홈페이지 상세 표시
     - 카카오맵 마커 표시 + 인포윈도우 클릭
     - 카카오 내비게이션 연결 (새 탭)

  ✅ 즐겨찾기
     - 카드에서 하트 버튼으로 토글 (❤️/🤍)
     - 마이페이지에서 즐겨찾기 목록 확인

  ✅ 리뷰
     - 관광지 상세 모달에서 별점(1~5) + 내용 작성
     - 비로그인도 리뷰 목록 조회 가능
     - 마이페이지에서 내 리뷰 확인 및 삭제

  ✅ 방문 기록
     - 관광지 카드 클릭 시 자동으로 저장 (로그인 상태일 때만)
     - 마이페이지에서 방문 기록 날짜 순으로 확인

  ✅ 다국어 지원
     - 한국어 / 영어 전환 (i18n.js)
     - 영어 선택 시 한국관광공사 영문 API (EngService2) 호출

----------------------------------------------------------------
  환경 변수 (.env 파일)
----------------------------------------------------------------

  프로젝트 루트(pom.xml 있는 폴더)에 .env 파일 생성 후 아래 내용 입력:

  TOUR_API_KEY=한국관광공사_한국어_API_키
  TOUR_API_KEY_EN=한국관광공사_영어_API_키
  DB_PASSWORD=MariaDB_비밀번호

  ※ spring-dotenv 라이브러리가 자동으로 읽어서 application.properties에 주입

----------------------------------------------------------------
  실행 방법
----------------------------------------------------------------

  1. MariaDB 실행 후 데이터베이스 생성:
     CREATE DATABASE projectDB CHARACTER SET utf8mb4;

  2. 프로젝트 루트에 .env 파일 생성 후 API 키, DB 비밀번호 입력

  3. 터미널에서 실행:
     mvn spring-boot:run

  4. 브라우저에서 접속:
     http://localhost:8080
